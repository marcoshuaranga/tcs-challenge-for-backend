# C4 Model — Order Processing Platform

Three levels of zoom: system context → containers → components (core/orders).

---

## Level 1 — System Context

Who uses the platform and what does it talk to?

```mermaid
C4Context
  title System Context — Order Processing Platform

  Person(customer, "Customer", "Creates orders and polls their status via the web app")
  Person(backoffice, "Backoffice Operator", "Views the full order list via the backoffice page")
  Person(developer, "Developer / Reviewer", "Reads the live OpenAPI spec via the api-docs endpoint")

  System(platform, "Order Processing Platform", "Registers orders, processes them asynchronously through a state machine, and maintains a full audit trail for every transition")

  System_Ext(aws, "AWS", "Lambda, API Gateway HTTP API, DynamoDB, SQS + DLQ, S3 + CloudFront, CloudWatch Logs")

  Rel(customer, platform, "Creates orders, checks status", "HTTPS / Bearer JWT")
  Rel(backoffice, platform, "Lists all orders", "HTTPS / Bearer JWT")
  Rel(developer, platform, "Reads OpenAPI docs", "HTTPS")
  Rel(platform, aws, "Deployed on", "IAM / AWS SDK")
```

## Level 2 — Containers

What processes / deployable units make up the platform?

```mermaid
C4Container
  title Container Diagram — Order Processing Platform

  Person(customer, "Customer", "Browser")
  Person(backoffice, "Backoffice Operator", "Browser")

  System_Boundary(platform, "Order Processing Platform") {
    Container(web, "web", "Astro + Tailwind + DaisyUI — S3 + CloudFront", "3 public pages: landing, customer order form, backoffice table. Pre-signed demo JWT in .env.example. No server-side rendering. Deployed as a static build to S3, served via CloudFront with a URL-rewrite Function.")
    Container(api, "orders-api", "Hono on Lambda (hono/aws-lambda)", "REST API — POST /orders, GET /orders/:id, GET /orders/:id/audit, POST /orders/:id/process, GET /orders. Bearer JWT middleware on /orders*.")
    Container(worker, "orders-worker", "Node.js Lambda (SQS event source)", "Async consumer. Runs the order state machine: PENDING → PROCESSING → COMPLETED | FAILED. Throws InvalidStateTransitionError (→ 409) if order is not in PENDING state.")
    Container(apidocs, "api-docs", "Hono on Lambda", "Generates OpenAPI 3.1 spec at request time from core/contracts Zod schemas and serves the Scalar UI.")
    ContainerDb(dynamo, "DynamoDB", "Single-table + GSI1", "Stores Order items (PK=ORDER#id, SK=#META) and AuditEntry items (SK=AUDIT#ts#ulid). GSI1 enables list-all without a scan.")
    ContainerQueue(queue, "orders-queue", "SQS Standard Queue", "Buffers ProcessOrder messages published by orders-api. At-least-once delivery; visibility timeout > worker processing time.")
    ContainerQueue(dlq, "orders-dlq", "SQS Dead-Letter Queue", "Receives messages after maxReceiveCount=3 retries. CloudWatch alarm monitors depth.")
  }

  Rel(customer, web, "Uses", "HTTPS")
  Rel(backoffice, web, "Uses", "HTTPS")
  Rel(web, api, "Calls REST API", "HTTPS / Bearer JWT")
  Rel(api, dynamo, "PutItem / GetItem / Query", "AWS SDK / dynamodb-toolbox")
  Rel(api, queue, "SendMessage (ProcessOrder)", "AWS SQS SDK")
  Rel(queue, worker, "Triggers per batch (batchSize=1)", "SQS event source mapping")
  Rel(worker, dynamo, "PutItem / GetItem / Query", "AWS SDK / dynamodb-toolbox")
  Rel(queue, dlq, "After maxReceiveCount=3", "SQS redrive policy")
```

## Level 3 — Components (core/orders)

What are the moving parts inside the shared core package?

```mermaid
C4Component
  title Component Diagram — core/orders

  Container_Ext(ordersApi, "orders-api", "Hono Lambda")
  Container_Ext(ordersWorker, "orders-worker", "Node.js Lambda")
  ContainerDb_Ext(dynamo, "DynamoDB", "Single-table + GSI1")
  ContainerQueue_Ext(sqsQueue, "SQS Queue", "orders-queue")

  Container_Boundary(core, "core/orders (src/)") {

    Component(compose, "composeOrders(env)", "Composition root — index.ts", "Factory called once per process. Reads USE_AWS_DYNAMO and USE_AWS_SQS flags and wires the matching adapter variants into the handler graph. Returns a fully-wired OrderAppService.")

    Component(appService, "OrderAppService", "Application facade", "Thin delegator: registerOrder → CreateOrderHandler, processOrder → ProcessOrderHandler, getOrder → GetOrderHandler, listOrders → ListOrdersHandler, getAuditTrail → GetOrderAuditHandler.")

    Component(createHandler, "CreateOrderHandler", "Command handler", "Validates input. Builds Order(PENDING). Saves. Records ORDER_CREATED audit entry. Publishes ProcessOrderMessage via MessagePublisherPort.")
    Component(processHandler, "ProcessOrderHandler", "Command handler", "Guards: throws InvalidStateTransitionError if order not PENDING (→ HTTP 409). Transitions PENDING→PROCESSING (audit). Calls PaymentGatewayPort.authorize. On approval: COMPLETED; on decline: FAILED (with reason). Writes audit on every transition.")
    Component(auditHandler, "RecordAuditEntryHandler", "Command handler", "Appends an AuditEntry via AuditRepositoryPort. Called explicitly by every handler that transitions state — no implicit listeners or DynamoDB Streams.")
    Component(queryHandlers, "Query Handlers", "GetOrderHandler · ListOrdersHandler · GetOrderAuditHandler", "Read-only. GetOrder and GetAudit use OrderRepositoryPort / AuditRepositoryPort. ListOrders queries GSI1 (constant partition key ORDERS).")

    Component(domain, "Domain", "Order aggregate + value objects + state machine", "Pure TypeScript — no framework imports. Order.create(), startProcessing(), complete(), fail(). Illegal transitions throw InvalidStateTransitionError (→ HTTP 409). Money validates positive amount + ISO-4217 currency.")

    Component(orderRepo, "OrderRepositoryPort", "DynamoOrderRepository | InMemoryOrderRepository", "save(order), findById(id), listAll(). Dynamo variant uses dynamodb-toolbox single-table.")
    Component(auditRepo, "AuditRepositoryPort", "DynamoAuditRepository | InMemoryAuditRepository", "append(entry), findByOrderId(id). Audit items keyed SK=AUDIT#ts#ulid under the same PK.")
    Component(publisher, "MessagePublisherPort", "SqsMessagePublisher | InMemoryMessagePublisher", "publishProcessOrder(orderId). SQS variant uses QUEUE_URL. In-memory variant shares the queue instance with the poll-loop in the combined local runtime (ADR-0010).")
    Component(gateway, "PaymentGatewayPort", "FakePaymentGateway", "authorize(order): approved when amount ≤ FAIL_ABOVE_AMOUNT, declined otherwise. Swappable for a real gateway adapter without domain changes (ADR-0013).")
  }

  Rel(ordersApi, compose, "Calls once at module load", "composeOrders(env)")
  Rel(ordersWorker, compose, "Calls once at module load", "composeOrders(env)")
  Rel(compose, appService, "Returns wired instance")

  Rel(appService, createHandler, "registerOrder →")
  Rel(appService, processHandler, "processOrder →")
  Rel(appService, queryHandlers, "getOrder / listOrders / getAuditTrail →")

  Rel(createHandler, domain, "Order.create(props)")
  Rel(processHandler, domain, "startProcessing() / complete() / fail(reason)")

  Rel(createHandler, auditHandler, "RecordAuditEntry(null → PENDING, ORDER_CREATED)")
  Rel(processHandler, auditHandler, "RecordAuditEntry per transition (PROCESSING_STARTED, COMPLETED, FAILED)")

  Rel(createHandler, publisher, "publishProcessOrder(orderId)")
  Rel(processHandler, gateway, "authorize(order)")

  Rel(orderRepo, dynamo, "GetItem / PutItem / Query GSI1", "dynamodb-toolbox")
  Rel(auditRepo, dynamo, "PutItem / Query by PK", "dynamodb-toolbox")
  Rel(publisher, sqsQueue, "SendMessage", "AWS SQS SDK")
```
